## 함수 객체의 내부 슬롯 `[[Environment]]`

- 함수는 자신이 정의된 환경, 즉 상위 스코프를 기억해야 한다.
  이를 위해 함수 정의가 평가되어 함수 객체를 생성할 때
  함수는 자신의 내부 슬롯 `[[Envirionment]]`에 자신이 정의된 환경, 즉 상위 스코프의 참조를 저장한다.
- 함수 자신의 내부 슬롯 `[[Environment]]`에 저장된 상위 스코프의 참조는
  현재 실행 중인 실행 컨텍스트의 렉시컬 환경이다.
  → 함수 정의가 평가되어 함수 객체를 생성하는 시점은 함수가 정의된 환경,
  즉 상위 함수가 평가 또는 실행되고 있는 시점이고,
  이때 실행 중인 실행 컨텍스트는 상위 함수의 실행 컨텍스트이기 때문이다.

<aside>
💡

함수 객체의 내부 슬롯 `[[Environment]]`에 저장된
현재 실행 중인 실행 컨텍스트의 렉시컬 환경의 참조가 바로 상위 스코프다.

함수 객체는 내부 슬롯 `[[Environment]]`에 저장한 렉시컬 환경의 참조,
즉 상위 스코프를 자신이 존재하는 한 기억한다.

⇒ 이것이 ‘렉시컬 스코프’의 실체!

</aside>

## 클로저

<aside>
💡

모든 함수는 자신의 상위 스코프를 기억한다.

따라서 함수를 어디서 호출하든 상관없이 함수는 언제나 자신이 기억하는 상위 스코프의 식별자를 참조할 수 있으며, 식별자에 바인딩된 값을 변경할 수도 있다.

→ 이러한 원리로 인해 외부 함수보다 중첩 함수가 더 오래 유지되는 경우 중첩 함수는 **이미 생명 주기가 종료한 외부 함수의 변수를 참조**할 수 있다. 이러한 중첩 함수를 **클로저**라고 한다.

외부 함수가 종료(콜스택에서 제거)되었다고 해서 그 함수의 렉시컬 환경까지 소멸하는 것이 아니다.

→ 중첩 함수에 의해 참조되고 있기 때문에! (참조되고 있는 메모리 공간은 가비지 컬렉션 대상이 아님)

</aside>

### 클로저가 아닌 경우

- 이론적으로 모든 함수는 상위 스코프를 기억하므로 모든 함수는 클로저이지만,
  일반적으로 상위 스코프의 어떤 식별자도 참조하지 않는 함수는 클로저가 아니다.
- 외부 함수보다 생명 주기가 짧은 중첩 함수는 클로저가 아니다.
  → 생명 주기가 종료된 외부 함수의 식별자를 참조할 수 있다는 클로저의 본질에 맞지 않음

  ```jsx
  function foo() {
    const x = 1;

    // bar는 클로저였지만 곧바로 소멸 -> 클로저 아님
    function bar() {
      console.log(x);
    }

    bar();
  }

  foo();
  ```

### 클로저가 되기 위한 조건

- 중첩 함수가 상위 스코프의 식별자를 참조
  - 자유 변수: 클로저에 의해 참조되는 상위 스코프의 변수
- 중첩 함수가 외부 함수보다 오래 생존

### 클로저의 활용

클로저는 상태를 안전하게 변경하고 유지하기 위해 사용
→ 상태가 의도치 않게 변경되지 않도록 상태를 안전하게 은닉하고 특정 함수에게만 상태 변경 허용

```jsx
// 즉시 실행 함수
const increase = (function () {
  let num = 0;

  return function () {
    return ++num;
  };
})();

console.log(increase()); // 1
console.log(increase()); // 2
console.log(increase()); // 3
```

<aside>
❓

**즉시 실행 함수(IIFE)를 외부에서 접근할 수 없는 이유?**

1. IIFE가 호출되는 즉시, 엔진은 콜 스택에 해당 함수의 실행 컨텍스트를 푸시하고 코드 실행
2. 함수 내부의 모든 코드가 실행 완료되면, 해당 실행 컨텍스트는 콜스택에서 pop
3. 내부 지역변수는 가비지 컬렉션 대상이 되어 메모리에서 제거

   → 외부에서 접근 불가

즉시 실행 함수는 익명함수 형태로 괄호 `()`로 둘러싸여 작성되므로 ‘함수 표현식’으로 평가되고, 이 표현식은 평가와 동시에 단 한 번 실행되어 함수 자체의 이름이나 내부 식별자를 전역 객체나 외부 환경에 등록하지 않음

IIFE는 자바스크립트 엔진이 보장하는 격리된 메모리 공간과 실행 환경을 강제로 생성하고 즉시 폐기하는 방식이기 때문에 외부 접근이 원천적으로 차단됨.

</aside>
